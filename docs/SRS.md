# Project_01
Software Requirements Specification (SRS)
1. Introduction

This document describes the functional and non-functional requirements of the system.
It defines what the system should do and how it should perform to meet user needs.

2. Purpose

The purpose of this document is to provide clear and complete system requirements for developers, testers, and stakeholders.

3. Scope

The system is designed to manage student records and provide user account services such as registration, login, and notifications.

4. Functional and Non-Functional Requirements
4.1 Improved System Requirements

The following requirements are revised versions of the original system requirements.

R1: User Interface Usability

The system shall provide a simple and clear interface that allows new users to complete basic tasks within five minutes without external help.

R2: Data Security

The system shall store student records in an encrypted database and allow access only to authorized users.

R3: System Performance

The system shall load the main dashboard within three seconds under normal network conditions.

R4: User Account Management

The system shall allow users to register using a valid email address and password.
The system shall allow users to log in using authenticated credentials.
The system shall allow users to update their personal profile information.

R5: Notification System

The system shall send email notifications to users for account registration, password reset, and important system updates.
